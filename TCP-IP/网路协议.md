###网络协议

```
三次握手
1、 第一次，传输文件名，接收方收到文件名，应答OK
2、 第二次，发送文件尺寸，接收方收到数据，应答OK
3、 第三次，传输文件内容，接收方接收数据完成后应答OK并表示文件内容接收成功
```

```
分层模型
	OSI/RM 理论标准
      应用层
      表示层
      传输层
      网络层
      数据链路层
      物理层

    TCP/IP 实际模型
      应用层 (FTP Telnet TFTP NFS)  应用程序
      传输层 (TCP UDP)				进程到进程
      网络层 ( ICMP IP IGMP )	主机到主机
      链路层 ( ARP RARP协议 )  	设备到设备
   			统称为TCP/IP协议蔟
```

```
链路层
MAC地址
	网卡 ： 以太网规定，连入网络的所有设备，都必须具有“网卡”接口  网卡的标识称为：物理地址（MAC地址）
    源地址， 目的地址

    ARP：通过IP找MAC
	RARP: 通过MAC找IP

	广播： 只有链路层，直接寻找目的MAC，导致广播风暴 （只在当前局域网有效)

MAC： 物理地址
IP ： 逻辑地址

网络层作用
	源IP ， 目的ip
	引入一个新地址： IP地址
	ICMP：ping
    IP：
    IGMP：
传输层作用：
传输层 ： 源端口， 目的端口
	TCP： 更安全
    UDP： 更快

应用层：
	FTP： 下载
    Telnet: 远程连接
```

```
网络通信条件:
	1） 网卡，MAC地址（不需要用户处理）
    2） 逻辑地址： IP地址（需要用户指定）
    3） 端口：
    		同一个系统，个程序只能绑定一个端口
            不同系统，同一端口对应的程序可能不一样
```
